## Signoff Gates
1. Coverage
- [ ] Dashboard, Students, Teachers, Enrollment, Reports, Operations are specified.
- [ ] Login/session support contract is specified.

2. Action-State Clarity
- [ ] Every critical action has clear enable/disable rules.
- [ ] Destructive actions are tier-confirmed and reason-coded.

3. Validation Clarity
- [ ] Field-level and policy-level validations are separated.
- [ ] Blocking vs warning validations are explicit.

4. UX State Completeness
- [ ] Empty/loading/error/success states are defined per major screen.
- [ ] Error states include user recovery paths.

5. Governance Alignment
- [ ] Screen spec aligns with Step 1 scope lock.
- [ ] Screen spec aligns with Step 2 To-Be flows.
- [ ] Audit touchpoints exist for all governed actions.
